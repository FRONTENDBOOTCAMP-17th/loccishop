## 3-1. 리뷰 목록에서 `innerHTML`에 사용자 데이터를 직접 넣으면 위험해요 — XSS 공격 (중요도: 높음)

### 문제

`src/pages/product/detail/handlers/initReviews.js` 46~71번째 줄에서 API로 받은 리뷰 데이터(`review.title`, `review.content`, `review.author`)를 `innerHTML`의 템플릿 리터럴에 직접 삽입하고 있어요. 악의적인 사용자가 리뷰에 `<script>` 태그나 이벤트 핸들러를 넣으면 다른 사용자의 브라우저에서 실행될 수 있어요.

### 해결 방법

`innerHTML` 대신 DOM 요소를 직접 생성하고 `textContent`로 텍스트를 삽입하면 안전해요.

**Step 1.** `initReviews.js`의 리뷰 목록 렌더링 부분을 수정하세요.

```js
// 수정 전 (src/pages/product/detail/handlers/initReviews.js 43~72번째 줄)
reviews.forEach((review) => {
    const li = document.createElement("li");
    li.className = "h-full";
    li.innerHTML = `
      <article class="h-full flex flex-col gap-3 shadow-[0_0_6px_0_rgba(0,0,0,0.05)] rounded-lg p-5 bg-rose-white">
        <div class="flex">
          ${"<img src='/src/assets/icon/star.svg' class='w-3 h-3' />".repeat(review.rating)}
        </div>
        <h3 class="review-title leading-6">${review.title ?? ""}</h3>
        <div class="flex justify-between text-xs font-medium">
          <span>${review.author}</span>
          <span>${review.createdAt.slice(0, 10)}</span>
        </div>
        <p class="text-sm leading-5 text-zambezi">${review.content}</p>
        <div class="flex gap-2">
          ${review.reviewImages
            .map(
              (src) =>
                `<img src="${src}" class="w-20 h-20 object-cover rounded-xl bg-cararra" />`,
            )
            .join("")}
        </div>
        <div class="flex justify-between text-xs mt-auto pt-3">
          <button type="button">이 제품을 추천합니다 (${review.recommendCount})</button>
          <button type="button" class="text-abbey/60 underline">이 리뷰 신고하기</button>
        </div>
      </article>
    `;
    reviewList.append(li);
  });

// 수정 후 — DOM 요소 직접 생성으로 XSS 방지
reviews.forEach((review) => {
  const li = document.createElement("li");
  li.className = "h-full";

  const article = document.createElement("article");
  article.className =
    "h-full flex flex-col gap-3 shadow-[0_0_6px_0_rgba(0,0,0,0.05)] rounded-lg p-5 bg-rose-white";

  // 별점
  const starsDiv = document.createElement("div");
  starsDiv.className = "flex";
  for (let i = 0; i < review.rating; i++) {
    const star = document.createElement("img");
    star.src = "/src/assets/icon/star.svg";
    star.className = "w-3 h-3";
    star.alt = "";
    star.setAttribute("aria-hidden", "true");
    starsDiv.append(star);
  }

  // 제목
  const title = document.createElement("h3");
  title.className = "review-title leading-6";
  title.textContent = review.title ?? "";   // ← textContent = 안전!

  // 작성자 + 날짜
  const metaDiv = document.createElement("div");
  metaDiv.className = "flex justify-between text-xs font-medium";
  const authorSpan = document.createElement("span");
  authorSpan.textContent = review.author;    // ← textContent = 안전!
  const dateSpan = document.createElement("span");
  dateSpan.textContent = review.createdAt.slice(0, 10);
  metaDiv.append(authorSpan, dateSpan);

  // 본문
  const content = document.createElement("p");
  content.className = "text-sm leading-5 text-zambezi";
  content.textContent = review.content;      // ← textContent = 안전!

  // 리뷰 이미지
  const imagesDiv = document.createElement("div");
  imagesDiv.className = "flex gap-2";
  review.reviewImages.forEach((src) => {
    const img = document.createElement("img");
    img.src = src;                           // ← src 속성으로 직접 설정
    img.className = "w-20 h-20 object-cover rounded-xl bg-cararra";
    img.alt = "리뷰 이미지";
    imagesDiv.append(img);
  });

  // 추천/신고 버튼
  const actionsDiv = document.createElement("div");
  actionsDiv.className = "flex justify-between text-xs mt-auto pt-3";
  const recommendBtn = document.createElement("button");
  recommendBtn.type = "button";
  recommendBtn.textContent = `이 제품을 추천합니다 (${review.recommendCount})`;
  const reportBtn = document.createElement("button");
  reportBtn.type = "button";
  reportBtn.className = "text-abbey/60 underline";
  reportBtn.textContent = "이 리뷰 신고하기";
  actionsDiv.append(recommendBtn, reportBtn);

  article.append(starsDiv, title, metaDiv, content, imagesDiv, actionsDiv);
  li.append(article);
  reviewList.append(li);
});
```

**Step 2.** 별점 영역에 접근성 속성을 추가하면 더 좋아요.

```js
// starsDiv 생성 후 추가
starsDiv.setAttribute("role", "img");
starsDiv.setAttribute("aria-label", `별점 5점 만점에 ${review.rating}점`);
```

> **쉽게 말하면:** `innerHTML`은 문을 활짝 열어놓는 것과 같아요 — 좋은 사람도 들어오지만 나쁜 사람도 들어올 수 있어요. `textContent`는 대화만 할 수 있는 인터폰과 같아서 안전해요.

## 3-2. 헤더 컴포넌트에 HTML 구조 문제가 3가지 있어요 (중요도: 높음)

### 문제

`src/components/header/header.js`에서:
1. `document.createElement("header")`로 `<header>` 요소를 만들어서 `<header id="header">` 안에 넣음 → 이중 중첩
2. `<ul>` 안에 `<li>` 없이 `<img>` 직접 사용 → HTML 표준 위반
3. 모든 아이콘 이미지에 `alt=""` → 접근성 부족

### 해결 방법

**Step 1.** `src/components/header/header.js` 파일 전체를 아래와 같이 수정하세요.

```js
// 수정 전 (src/components/header/header.js 전체)
export function renderHeader() {
  const header = document.createElement("header");
  header.className = "bg-header-footer shadow p-4 w-full h-16";

  header.innerHTML = `
    <div class="header-inner h-full flex items-center justify-between max-inline-360 mx-auto">
      <img src="/src/assets/icon/Hamburger.svg" alt="" class="hamburger-btn cursor-pointer" />
      <img src="/src/assets/logo/Loccitane.svg" alt="" />
      <ul class="flex gap-4">
        <img src="/src/assets/icon/myPage.svg" alt="" />
        <img src="/src/assets/icon/heart-empty.svg" alt="" />
        <img src="/src/assets/icon/cart.svg" alt="" />
      </ul>
    </div>
  `;

  return header;
}

// 수정 후
export function renderHeader() {
  // ① <header> 대신 <nav>를 만들어서 중첩 방지
  const nav = document.createElement("nav");
  nav.className = "bg-spring-wood shadow p-4 w-full h-16";

  nav.innerHTML = `
    <div class="h-full flex items-center justify-between max-w-sm mx-auto">
      <!-- ② 햄버거 아이콘을 <button>으로 감싸서 키보드 접근 가능 -->
      <button type="button" class="hamburger-btn cursor-pointer" aria-label="메뉴 열기">
        <img src="/src/assets/icon/Hamburger.svg" alt="" aria-hidden="true" />
      </button>

      <!-- ③ 로고에 의미 있는 alt 텍스트 추가 -->
      <a href="/">
        <img src="/src/assets/logo/Loccitane.svg" alt="L'Occitane 홈으로" />
      </a>

      <!-- ④ <ul> 안에 <li> 추가, 각 아이콘에 alt 텍스트 추가 -->
      <ul class="flex gap-4 list-none">
        <li><a href="/src/pages/login/index.html"><img src="/src/assets/icon/myPage.svg" alt="마이페이지" /></a></li>
        <li><a href="#"><img src="/src/assets/icon/heart-empty.svg" alt="위시리스트" /></a></li>
        <li><a href="#"><img src="/src/assets/icon/cart.svg" alt="장바구니" /></a></li>
      </ul>
    </div>
  `;

  return nav;
}
```

**Step 2.** `src/js/main.js`에서 햄버거 버튼 선택자를 확인하세요.

현재 `main.js`에서 `.hamburger-btn` 클래스로 선택하고 있는데, 수정 후에도 `<button>` 태그에 이 클래스가 있으므로 **기존 코드를 수정할 필요 없어요!**

```js
// main.js — 이 부분은 수정 불필요
const hamburgerBtn = document.querySelector(".hamburger-btn");
if (hamburgerBtn) {
  hamburgerBtn.addEventListener("click", () => {
    menuDrawer.open();
  });
}
```

> **쉽게 말하면:**
> - `<header>` 중첩 → 집 안에 또 집을 짓는 것처럼 이상해요. 집(header) 안에는 방(nav)을 넣는 게 자연스러워요.
> - `<ul>` 안에 `<li>` 없음 → 번호가 매겨진 서류철에 서류를 폴더 없이 넣는 것과 같아요.
> - `alt=""` → 눈을 감고 웹사이트를 사용한다고 상상해 보세요. 뭐가 뭔지 알려줘야 해요.

## 3-3. `bg-header-footer` 색상이 정의되어 있지 않아요 (중요도: 높음)

### 문제

`src/components/header/header.js` 3번째 줄과 `src/components/footer/footer.js` 4번째 줄에서 `bg-header-footer`라는 Tailwind 클래스를 사용하는데, `src/styles/style.css`의 `@theme` 블록에 `--color-header-footer`가 정의되어 있지 않아요.

### 해결 방법

두 가지 방법 중 하나를 선택하세요.

**방법 A: 이미 있는 색상 사용 (가장 간단!)**

```js
// 수정 전 (src/components/header/header.js 3번째 줄)
header.className = "bg-header-footer shadow p-4 w-full h-16";

// 수정 후 — spring-wood 색상 사용 (가장 간단한 방법!)
header.className = "bg-spring-wood shadow p-4 w-full h-16";
// 만약 3-2를 먼저 수정했다면 nav.className으로 변경
```

```js
// 수정 전 (src/components/footer/footer.js 3~4번째 줄)
footer.className =
  "w-full bg-header-footer pt-10 pb-8 border-t border-gray-200";

// 수정 후
footer.className =
  "w-full bg-spring-wood pt-10 pb-8 border-t border-gray-200";
```

**방법 B: style.css에 새 색상 정의**

디자인 시안에서 헤더/푸터 배경색이 따로 있다면:

```css
/* src/styles/style.css @theme 블록에 추가 */
@theme {
  /* 기존 색상들... */

  /* 헤더/푸터 배경색 */
  --color-header-footer: #fbf9f6;    /* 디자인 시안에 맞는 색상 코드로! */
}
```

> **쉽게 말하면:** "이 셔츠를 파란색으로 염색해주세요"라고 했는데 파란 염료가 없으면 어떻게 될까요? 염료 상자에 있는 색 이름을 써주거나, 파란 염료를 새로 사야 해요!

## 3-5. 햄버거 메뉴 아이콘이 `<img>`라서 키보드로 접근할 수 없어요 (중요도: 중간)

### 문제

`src/components/header/header.js` 7번째 줄에서 `<img>`를 직접 클릭 대상으로 사용하고 있어요.

### 해결 방법

이 항목은 **3-2의 수정에 포함**되어 있어요. 3-2를 수정하면 자동으로 해결돼요!

핵심은 `<img>` 태그를 `<button>` 태그로 감싸는 것이에요:

```html
<!-- 수정 전 -->
<img src="/src/assets/icon/Hamburger.svg" alt="" class="hamburger-btn cursor-pointer" />

<!-- 수정 후 -->
<button type="button" class="hamburger-btn cursor-pointer" aria-label="메뉴 열기">
  <img src="/src/assets/icon/Hamburger.svg" alt="" aria-hidden="true" />
</button>
```

**왜 `<button>`이어야 할까?**

| 기능 | `<img>` | `<button>` |
|------|---------|------------|
| 마우스 클릭 | O | O |
| Tab 키로 이동 | X | O |
| Enter 키로 실행 | X | O |
| 스크린리더가 "버튼"으로 읽기 | X | O |

> **쉽게 말하면:** `<img>`는 사진이에요. 사진을 아무리 터치해도 문이 열리지 않죠. `<button>`은 초인종이에요. 누르면 문이 열려요!

## 3-6. `initReviews`에서 리뷰를 999개 한꺼번에 불러와요 (중요도: 중간)

### 문제

`src/pages/product/detail/handlers/initReviews.js` 6번째 줄에서 `limit: 999`로 모든 리뷰를 한 번에 요청해요.

### 해결 방법

**Step 1.** `limit`을 적절한 수(예: 10)로 변경하세요.

```js
// 수정 전 (src/pages/product/detail/handlers/initReviews.js 5~9번째 줄)
const { reviews, meta } = await fetchProductReviews(productId, {
  page: 1,
  limit: 999,
  sort,
});

// 수정 후
const { reviews, meta } = await fetchProductReviews(productId, {
  page: 1,
  limit: 10,
  sort,
});
```

**Step 2.** 별점 바 퍼센트 계산에 주의하세요!

현재 코드는 `reviews` 배열의 길이를 기반으로 퍼센트를 계산하고 있어요:

```js
// 현재 코드 (30~37번째 줄)
const percent = reviews.length > 0 ? Math.round((count / reviews.length) * 100) : 0;
```

`limit: 10`으로 바꾸면 10개 기준으로만 퍼센트가 계산돼요. 전체 리뷰 기준으로 계산하려면:

```js
// 수정 후 — 전체 리뷰 수 기준으로 계산
const percent = total > 0 ? Math.round((count / total) * 100) : 0;
```

단, 이 방법은 API에서 별점별 개수를 제공해야 정확해요. API 응답에 별점 분포 데이터가 없다면 현재 방식을 유지하되 `limit`만 조절하는 것도 괜찮아요.

**Step 3.** (선택) "리뷰 더보기" 버튼에 다음 페이지 로딩 기능을 추가하면 완성이에요!

```js
// 현재 "리뷰 더보기" 버튼은 만들어져 있지만 기능이 없어요.
// 아래처럼 페이지 변수를 관리하면 돼요:
let currentPage = 1;

// "리뷰 더보기" 클릭 시
moreReviewBtn.addEventListener("click", async () => {
  currentPage++;
  const { reviews: moreReviews } = await fetchProductReviews(productId, {
    page: currentPage,
    limit: 10,
    sort: "latest",
  });
  // moreReviews를 기존 리스트에 추가
});
```

> **쉽게 말하면:** 도서관에서 책을 빌릴 때 한 번에 999권을 가져오는 대신, 10권씩 빌려서 읽고, 다 읽으면 또 빌리는 게 효율적이에요!

```js
// 수정 전 (renderProductMain.js 48~54번째 줄)
dotsContainer.innerHTML = allImages.map((_, i) => `
  <button class="dot w-2 h-2 rounded-full ${i === 0 ? "bg-ferra" : "bg-stone-300"}" data-index="${i}"></button>
`).join("");

// 수정 후
dotsContainer.innerHTML = "";
allImages.forEach((_, i) => {
  const dot = document.createElement("button");
  dot.className = `dot w-2 h-2 rounded-full ${i === 0 ? "bg-ferra" : "bg-stone-300"}`;
  dot.dataset.index = i;
  dot.type = "button";
  dot.setAttribute("aria-label", `${i + 1}번째 이미지로 이동`);
  dotsContainer.append(dot);
});
```

> **쉽게 말하면:** 3-1과 같은 원리예요. `innerHTML`은 "적힌 대로 HTML을 만들어!"이고, DOM API는 "내가 하나씩 안전하게 조립할게!"예요. 이미지 URL도 신뢰할 수 없는 경우가 있으니 습관적으로 안전한 방법을 쓰는 것이 좋아요.

## 3-8. `product-card-list.js`에서 `resize` 이벤트 리스너가 계속 쌓여요 (중요도: 중간)

### 문제

`src/components/ui/product-card-list.js` 92~93번째 줄에서 `renderProductCards`가 호출될 때마다 새 resize 리스너가 추가되지만, 이전 리스너는 제거되지 않아요.

### 해결 방법

**Step 1.** 파일 최상단(import 아래)에 이전 핸들러를 저장할 변수를 추가하세요.

```js
// 수정 전 (src/components/ui/product-card-list.js 1번째 줄)
import { createProductCard } from "/src/components/ui/product-card.js";

// 수정 후 — 이전 핸들러 저장 변수 추가
import { createProductCard } from "/src/components/ui/product-card.js";

let prevCursorHandler = null;
let prevWidthHandler = null;
```

**Step 2.** `renderProductCards` 함수 안에서 기존 리스너를 제거한 후 새로 등록하세요.

```js
// 수정 전 (92~93번째 줄)
window.addEventListener("resize", updateCursor);
window.addEventListener("resize", updateCardWidths);

// 수정 후 — 기존 리스너 제거 후 새로 등록

window.addEventListener("resize", updateCardWidths);
```

> **쉽게 말하면:** 알람 앱에서 "내일 7시" 알람을 설정하는데, 버튼을 누를 때마다 알람이 추가돼요. 3번 눌렀으면 7시에 알람이 3번 울려요! 새 알람을 설정하기 전에 이전 알람을 지우면 돼요.

## 3-9. CSS에서 Tailwind로 대체할 수 있는 커스텀 스타일이 있어요 (중요도: 낮음)

### 문제

`src/components/ui/product-card-list.js`에서 `container.style.cursor`로 인라인 스타일을 설정하고 있어요.

### 해결 방법

**Step 1.** `updateCursor` 함수에서 인라인 스타일을 Tailwind 클래스로 변경하세요.

```js
// 수정 전 (src/components/ui/product-card-list.js 19~25번째 줄)
function updateCursor() {
  if (container.scrollWidth > container.clientWidth) {
    container.style.cursor = "grab";
  } else {
    container.style.cursor = "default";
  }
}

// 수정 후 — Tailwind 클래스 사용
function updateCursor() {
  if (container.scrollWidth > container.clientWidth) {
    container.classList.add("cursor-grab");
    container.classList.remove("cursor-default");
  } else {
    container.classList.remove("cursor-grab");
    container.classList.add("cursor-default");
  }
}
```

**Step 2.** mousedown 이벤트에서도 변경하세요.

```js
// 수정 전 (38번째 줄)
container.style.cursor = "grabbing";

// 수정 후
container.classList.remove("cursor-grab");
container.classList.add("cursor-grabbing");
```

**Step 3.** mouseleave와 mouseup에서는 `updateCursor()`를 호출하니까 자동으로 정리돼요. 다만 `cursor-grabbing`도 제거해야 해요:

```js
// updateCursor 함수에 추가
function updateCursor() {
  container.classList.remove("cursor-grabbing");  // ← 추가
  if (container.scrollWidth > container.clientWidth) {
    container.classList.add("cursor-grab");
    container.classList.remove("cursor-default");
  } else {
    container.classList.remove("cursor-grab");
    container.classList.add("cursor-default");
  }
}
```

> **쉽게 말하면:** Tailwind 프로젝트에서 `style="..."`를 쓰는 것은, 유니폼이 있는 학교에서 혼자 사복을 입고 온 것과 비슷해요. 같은 규칙(Tailwind)을 따르면 코드가 일관적이에요!
