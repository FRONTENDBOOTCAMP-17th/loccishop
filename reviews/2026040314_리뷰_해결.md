## 3-1. `style.css` — 잘못된 색상 코드 (중요도: 높음)

### 문제

`src/css/style.css` 16번째 줄에서 HEX 색상 코드가 5자리여서 유효하지 않아요.

### 해결 방법

**Step 1.** `src/css/style.css`를 열고 16번째 줄을 수정하세요.

```css
/* 수정 전 (src/css/style.css 16번째 줄) */
--color-empress: #7E717;

/* 수정 후 — 의도한 색상에 맞게 6자리로 수정 */
--color-empress: #7E7177;
```

HEX 색상 코드의 구조를 알면 쉬워요:
- `#7E7177` → `7E` = 빨강, `71` = 초록, `77` = 파랑
- 각 2자리가 한 색상 채널을 담당해요 (00~FF, 즉 0~255)
- 그래서 반드시 **6자리**(또는 축약형 3자리)여야 해요

원래 의도한 색상이 무엇인지 디자인 시안을 확인해서 정확한 값을 넣어주세요.

> **쉽게 말하면:** 전화번호가 한 자리 빠진 것처럼, 색상 코드도 자릿수가 맞아야 제대로 연결돼요!

## 3-2. CSS 변수 이름에 하이픈이 두 개 (중요도: 높음)

### 문제

`feature/#5-login` 브랜치의 `src/css/style.css`에서 `--color--grey-99`(하이픈 두 개)가 있고, 아래에 `--color-grey-99`(하이픈 한 개)도 있어서 같은 값의 변수가 두 개예요.

### 해결 방법

**Step 1.** `src/css/style.css`에서 `--color--grey-99` 줄을 찾아 삭제하세요.

```css
/* 수정 전 */
@theme {
  /* 로그인 배경1(전체배경) */
  --color--grey-99: #fffefa;        /* ← 이 줄 삭제! */

  /* ... 다른 변수들 ... */

  /* 로그인 input */
  --color-grey-99: #fffefa;         /* ← 이 줄만 유지 */
}

/* 수정 후 */
@theme {
  /* 삭제됨: --color--grey-99 */

  /* ... 다른 변수들 ... */

  /* 로그인 input 배경색 */
  --color-grey-99: #fffefa;
}
```

**Step 2.** HTML에서 혹시 `bg--grey-99`(하이픈 두 개)를 사용하고 있는 곳이 있는지 확인하고, 있다면 `bg-grey-99`로 수정하세요.

> **쉽게 말하면:** 같은 사람한테 이름표를 두 개 붙인 거예요. 하나만 있으면 돼요!

## 3-3. 같은 색상값이 다른 이름으로 중복 (중요도: 중간)

### 문제

`--color-woody-brown`(develop)과 `--color-brand-woody`(feature/#5-login)가 같은 색상(`#3f2b2e`)이에요.

### 해결 방법

**Step 1.** 팀원들과 사용할 이름 하나를 정하세요.

```css
/* 방법 A — woody-brown으로 통일 */
@theme {
  --color-woody-brown: #3f2b2e;     /* ← 유지 */
  /* --color-brand-woody: #3f2b2e;     ← 삭제 */
}

/* 방법 B — brand-woody로 통일 */
@theme {
  /* --color-woody-brown: #3f2b2e;     ← 삭제 */
  --color-brand-woody: #3f2b2e;     /* ← 유지 */
}
```

**Step 2.** HTML에서 삭제한 이름을 사용하고 있는 곳을 모두 찾아 변경하세요.

예를 들어 `brand-woody`를 삭제하고 `woody-brown`으로 통일한다면:

```html
<!-- 수정 전 -->
<h1 class="text-brand-woody">로그인 / 회원가입</h1>

<!-- 수정 후 -->
<h1 class="text-woody-brown">로그인 / 회원가입</h1>
```

모든 파일에서 바꿔야 하니까 에디터의 **전체 찾기 & 바꾸기** 기능을 사용하면 편해요:
- VS Code: `Ctrl + Shift + H` (Mac: `Cmd + Shift + H`)
- 찾기: `brand-woody` → 바꾸기: `woody-brown`

> **쉽게 말하면:** 같은 물건에 "빨간 펜"과 "레드 펜"이라는 이름이 두 개 있으면 헷갈리잖아요. 하나로 통일!

## 3-4. 로그인 모달 — 모바일 반응형 (중요도: 중간)

### 문제

`feature/#5-login` 브랜치의 `src/pages/login.html`에서 모달 너비가 `w-200`(800px), 내부 콘텐츠가 `w-160`(640px)으로 고정되어 있어서 모바일 화면(360~430px)에서 넘쳐요.

### 해결 방법

**Step 1.** 모달 컨테이너의 너비를 반응형으로 변경하세요.

```html
<!-- 수정 전 (login.html) -->
<div class="relative pr-2.5 bg-grey-99 z-10 w-200 h-fit">

<!-- 수정 후 -->
<div class="relative px-2.5 bg-grey-99 z-10 w-full max-w-[800px] mx-4 h-fit">
```

변경 내용:
- `w-200` → `w-full max-w-[800px]` : 화면 꽉 차되 최대 800px
- `pr-2.5` → `px-2.5` : 좌우 패딩 동일하게
- `mx-4` 추가 : 모바일에서 좌우 여백 확보

**Step 2.** 내부 콘텐츠 영역도 반응형으로 변경하세요.

```html
<!-- 수정 전 -->
<div class="my-[71.99px] w-160 mx-9.25 flex flex-col justify-start items-start">

<!-- 수정 후 -->
<div class="my-10 md:my-[72px] w-full px-6 md:px-9 flex flex-col justify-start items-start">
```

변경 내용:
- `w-160` → `w-full` : 부모 너비에 맞춤
- `mx-9.25` → `px-6 md:px-9` : 모바일에서 작은 패딩, 데스크탑에서 큰 패딩
- `my-[71.99px]` → `my-10 md:my-[72px]` : 모바일에서 적은 여백

**Step 3.** 폼 영역의 고정 너비도 수정하세요.

```html
<!-- 수정 전 -->
<div class="px-10 py-4 bg-grey-96 w-181.5 flex flex-col gap-4">

<!-- 수정 후 -->
<div class="px-6 md:px-10 py-4 bg-grey-96 w-full flex flex-col gap-4">
```

> **쉽게 말하면:** 큰 액자를 작은 벽에 걸면 삐져나오잖아요. "벽 크기에 맞추되 최대 이만큼"이라고 해주는 거예요!

## 3-5. 로그인 폼 — form action 처리 (중요도: 중간)

### 문제

`feature/#5-login` 브랜치의 `src/pages/login.html` 41번째 줄에서 `<form action="">`으로 비어 있어요.

### 해결 방법

**Step 1.** `<form>` 태그를 수정하세요.

```html
<!-- 수정 전 (login.html 41번째 줄) -->
<form action="">

<!-- 수정 후 -->
<form id="loginForm">
```

**Step 2.** 로그인 JS 파일을 새로 만드세요. `src/js/login.js`:

```js
// src/js/login.js — 새 파일
import "../css/style.css";

const loginForm = document.getElementById("loginForm");

loginForm.addEventListener("submit", (e) => {
  e.preventDefault(); // 페이지 새로고침 방지!

  const email = document.getElementById("email").value;
  const password = document.getElementById("password").value;

  console.log("로그인 시도:", email);
  // TODO: 여기서 API 호출
});
```

**Step 3.** login.html의 `<script>` 태그를 수정하세요.

```html
<!-- 수정 전 -->
<script type="module" src="/src/js/main.js"></script>

<!-- 수정 후 -->
<script type="module" src="/src/js/login.js"></script>
```

`e.preventDefault()`가 핵심이에요. 이게 없으면 폼을 제출할 때 **페이지가 새로고침**되어서 사용자가 입력한 내용이 다 사라져요.

> **쉽게 말하면:** 편지를 보내기 전에 "잠깐! 내가 직접 배달할게"라고 우체통한테 말하는 거예요. 안 그러면 우체통이 알아서 보내버려요(페이지 새로고침).

## 3-6. 로그인 페이지 `<body>` 스타일 통일 (중요도: 낮음)

### 문제

index.html과 login.html의 `<body>` 스타일이 달라요.

### 해결 방법

**Step 1.** 두 가지 방법 중 하나를 선택하세요.

**방법 A — `@layer base` 활용 (추천)**

index.html의 `<body>` 클래스를 제거하고, `@layer base`에서 통합 관리하세요.

```html
<!-- index.html — 수정 전 -->
<body class="bg-gray-50 text-gray-900">

<!-- index.html — 수정 후 (클래스 제거) -->
<body>
```

`style.css`의 `@layer base`가 이미 body 스타일을 정의하고 있으므로 클래스가 필요 없어요:
```css
@layer base {
  body {
    @apply bg-[var(--color-spring-wood)];
    @apply text-[var(--color-woody-brown)];
  }
}
```

**방법 B — 모든 페이지에 같은 클래스 적용**

login.html의 `<body>`에도 같은 클래스를 추가하세요.
```html
<body class="bg-gray-50 text-gray-900">
```

> **쉽게 말하면:** 방마다 벽지가 다르면 어색하잖아요. 집 전체 벽지를 한 번에 정하면 편해요!

## 3-8. 빈 컴포넌트 파일 — header.js, footer.js (중요도: 낮음)

### 문제

`12-feat-header-footer` 브랜치의 `src/components/header.js`와 `footer.js`가 0바이트 빈 파일이에요.

### 해결 방법

**Step 1.** `src/components/header.js`에 기본 구조를 작성하세요.

```js
// src/components/header.js
export function createHeader() {
  return `
    <nav class="bg-white shadow-md">
      <div class="container mx-auto px-4 py-3 flex items-center justify-between">
        <a href="/">
          <img src="/src/assets/icon/logo.svg" alt="L'OcciShop 로고" class="h-8" />
        </a>
        <ul class="flex gap-6 text-sm">
          <li><a href="/" class="hover:text-ferra">홈</a></li>
          <li><a href="#" class="hover:text-ferra">상품</a></li>
          <li><a href="#" class="hover:text-ferra">장바구니</a></li>
        </ul>
      </div>
    </nav>
  `;
}
```

**Step 2.** `src/components/footer.js`에도 기본 구조를 작성하세요.

```js
// src/components/footer.js
export function createFooter() {
  return `
    <div class="bg-woody-brown text-merino py-8 mt-12">
      <div class="container mx-auto px-4 text-center text-sm">
        <p>&copy; 2026 L'OcciShop Clone Project</p>
      </div>
    </div>
  `;
}
```

**Step 3.** `src/js/main.js`에서 불러와 사용하세요.

```js
// src/js/main.js
import "../css/style.css";
import { createHeader } from "../components/header.js";
import { createFooter } from "../components/footer.js";

const headerEl = document.getElementById("header");
if (headerEl) {
  headerEl.innerHTML = createHeader();
}

const footerEl = document.getElementById("footer");
if (footerEl) {
  footerEl.innerHTML = createFooter();
}
```

> **쉽게 말하면:** 빈 서랍장을 배달받은 것과 같아요. 물건을 넣어야 의미가 있어요!

## 3-9. 아포스트로피 오타 (중요도: 낮음)

### 문제

`12-feat-header-footer` 브랜치의 `index.html`에서 `L'OcciShop`이 `L`OcciShop`으로 되어 있어요.

### 해결 방법


> **쉽게 말하면:** 브랜드 이름의 철자가 틀리면 공식적이지 않아 보여요. 이력서에 회사 이름 틀리는 것과 같아요!
