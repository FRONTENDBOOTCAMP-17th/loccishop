## 3-1. 상품 목록 페이지가 빌드 설정에 빠져 있어요 (중요도: 높음)

### 문제

`src/pages/product/list/index.html`이 `vite.config.js`의 빌드 설정에 포함되어 있지 않아요. 배포 시 이 페이지가 누락돼요.

### 해결 방법

**Step 1.** `vite.config.js`에 상품 목록 페이지를 추가하세요.

```js
// 수정 전 (vite.config.js 14~22번째 줄)
input: {
  main: resolve(__dirname, "index.html"),
  productDetail: resolve(
    __dirname,
    "src/pages/product/detail/index.html",
  ),
  login: resolve(__dirname, "src/pages/login/index.html"),
  signup: resolve(__dirname, "src/pages/signup/index.html"),
},

// 수정 후 — productList 한 줄 추가
input: {
  main: resolve(__dirname, "index.html"),
  productDetail: resolve(
    __dirname,
    "src/pages/product/detail/index.html",
  ),
  productList: resolve(
    __dirname,
    "src/pages/product/list/index.html",
  ),
  login: resolve(__dirname, "src/pages/login/index.html"),
  signup: resolve(__dirname, "src/pages/signup/index.html"),
},
```

**Step 2.** 제대로 추가되었는지 빌드로 확인해보세요:

```bash
npm run build
```

에러 없이 완료되고, `dist/` 폴더에 상품 목록 페이지가 포함되어 있으면 성공이에요!

> **쉽게 말하면:** Vite에게 "이 페이지도 배포할 때 포함시켜줘!"라고 알려주는 거예요. 목록에 없으면 Vite가 모르니까 빌드에서 빠져요.

## 3-3. 세로형 상품 카드에서 할인율이 0일 때 "0%"가 표시돼요 (중요도: 높음)

### 문제

`product-card.js`의 세로형 레이아웃에서 `discountRate`가 `0`이어도 "0%"가 화면에 표시돼요. `discountPrice`가 `null`이면 에러가 나요. (`product-card.js` 185~193번째 줄)

### 해결 방법

**Step 1.** `src/components/ui/product-card.js`의 세로형 가격 영역을 수정하세요.

```js
// 수정 전 (product-card.js 185~194번째 줄)
const discountRateEl = document.createElement("span");
discountRateEl.textContent = `${discountRate}%`;
discountRateEl.className = "text-sm text-burnt-orange font-semibold";

const discountPriceEl = document.createElement("span");
discountPriceEl.textContent = `${discountPrice.toLocaleString()}원`;
discountPriceEl.className = "text-sm text-woody-brown";

discountWrapper.append(discountRateEl, discountPriceEl);
priceRow.append(originalPriceEl, discountWrapper);

// 수정 후 — 가로형과 동일한 패턴 적용
if (discountRate) {
  const discountRateEl = document.createElement("span");
  discountRateEl.textContent = `${discountRate}%`;
  discountRateEl.className = "text-sm text-burnt-orange font-semibold";
  discountWrapper.append(discountRateEl);
}

const discountPriceEl = document.createElement("span");
discountPriceEl.textContent = `${(discountPrice ?? originalPrice).toLocaleString()}원`;
discountPriceEl.className = "text-sm text-woody-brown";
discountWrapper.append(discountPriceEl);

priceRow.append(originalPriceEl, discountWrapper);
```

**핵심 변경:**
1. `if (discountRate)` — 할인율이 `0`이면 할인율 표시를 건너뛰어요
2. `discountPrice ?? originalPrice` — `discountPrice`가 `null`이면 `originalPrice`를 대신 사용해요 (`??`는 **널 병합 연산자**라고 해요)

**Step 2.** 할인 없는 상품일 때 원가의 취소선도 숨기면 더 깔끔해요:

```js
// 추가 개선: 할인이 없으면 원가 취소선도 숨기기
const originalPriceEl = document.createElement("span");
originalPriceEl.textContent = `${originalPrice.toLocaleString()}원`;
originalPriceEl.className = "text-xs text-empress line-through";

// 할인이 없으면 원가 취소선 숨기기
if (!discountRate) {
  originalPriceEl.classList.add("hidden");
}
```

> **쉽게 말하면:** "이 상품은 50% 할인!"이라고 표시하면 좋지만, "이 상품은 0% 할인!"이라고 표시하면 이상하잖아요. 할인이 없으면 그냥 가격만 보여주는 게 자연스러워요.

## 3-4. `renderProductCards` 함수가 두 곳에 중복돼요 (중요도: 중간)

### 문제

`src/components/ui/product-card-list.js`에 이미 `renderProductCards` 함수가 있는데, `src/pages/product/list/index.js`에서 같은 함수를 다시 작성했어요.

### 해결 방법

**Step 1.** `src/pages/product/list/index.js`를 수정하세요.

```js
// 수정 전 (src/pages/product/list/index.js 전체)
import { createProductCard } from "/src/components/ui/product-card.js";

const productDataList = [
  // ... 상품 데이터 ...
];

const container = document.querySelector("#product-list-container");

function renderProductCards(products) {    // ← 중복 함수
  if (!container) return;
  container.innerHTML = "";
  products.forEach((product) => {
    const cardElement = createProductCard(product);
    container.appendChild(cardElement);
  });
}

renderProductCards(productDataList);

// 수정 후 — product-card-list.js를 import해서 사용
import { renderProductCards } from "/src/components/ui/product-card-list.js";

const productDataList = [
  // ... 상품 데이터 (그대로 유지) ...
];

renderProductCards(productDataList);
```

이렇게 하면:
- `createProductCard` import도 필요 없어요 (`product-card-list.js`가 내부적으로 import하니까)
- `container` 변수 선언도 필요 없어요
- **중복 코드 15줄이 2줄로** 줄어들어요!

> **쉽게 말하면:** 이미 만들어둔 도구(product-card-list.js)가 있으니까 새로 만들지 말고 가져다 쓰는 거예요. DRY 원칙(Don't Repeat Yourself)이라고 해요 — "같은 코드를 반복하지 마세요!"

## 3-5. 상품 목록 페이지에 `<main>` 태그가 없어요 (중요도: 중간)

`src/pages/product/list/index.html`에 `<main>` 태그가 없어서 시맨틱 구조와 접근성이 부족해요.

### 해결 방법

**Step 1.** `<body>` 바로 안쪽에 `<main>` 태그로 콘텐츠를 감싸세요.

```html
<!-- 수정 전 (src/pages/product/list/index.html 10~101번째 줄) -->
<body>
  <section class="relative w-full h-119 ...">
    <!-- 히어로 배너 -->
  </section>
  <div class="px-[232.5px] pt-10 flex flex-col">
    <!-- 콘텐츠 -->
  </div>
</body>

<!-- 수정 후 — <main>으로 감싸기 -->
<body>
  <header id="header"></header>

  <main>
    <section class="relative w-full h-119 ...">
      <!-- 히어로 배너 -->
    </section>
    <div class="px-[232.5px] pt-10 flex flex-col">
      <!-- 콘텐츠 -->
```

## 3-6. 상품 목록 페이지의 패딩이 고정값이라 작은 화면에서 깨져요 (중요도: 중간)

### 문제

`px-[232.5px]`은 고정 패딩이라 좁은 화면에서 콘텐츠가 잘려요. (`src/pages/product/list/index.html` 20번째 줄)

### 해결 방법

**Step 1.** 고정 패딩을 반응형 패딩으로 바꾸세요.

```html
<!-- 수정 전 (src/pages/product/list/index.html 20번째 줄) -->
<div class="px-[232.5px] pt-10 flex flex-col">

<!-- 수정 후 — 반응형 패딩 -->
<div class="px-6 md:px-16 lg:px-32 pt-10 flex flex-col max-w-7xl mx-auto w-full">
```

**각 클래스가 하는 일:**
- `px-6`: 모바일에서 좌우 24px 패딩 (기본)
- `md:px-16`: 태블릿(768px 이상)에서 좌우 64px
- `lg:px-32`: 데스크톱(1024px 이상)에서 좌우 128px
- `max-w-7xl`: 최대 너비 1280px — 아주 큰 모니터에서도 너무 넓어지지 않게
- `mx-auto`: 가운데 정렬
- `w-full`: 가로 전체 채우기

**Step 2.** 카테고리 리스트의 `px-13`도 비슷하게 조정하세요.

```html
<!-- 수정 전 -->
<ul class="flex gap-1.5 py-1 px-13">

<!-- 수정 후 -->
<ul class="flex gap-1.5 py-1 flex-wrap">
```

`flex-wrap`을 추가하면 화면이 좁을 때 카테고리 버튼이 **자동으로 다음 줄로 넘어가요**.

> **쉽게 말하면:** 넓은 책상에서만 작업할 수 있게 만든 것을, 좁은 책상에서도 작업할 수 있게 바꾸는 거예요.

## 3-7. 카테고리 링크들의 Tailwind 클래스가 6번 반복돼요 (중요도: 중간)

### 문제

6개의 카테고리 `<a>` 태그에 동일한 긴 클래스가 반복되고, `text-[18px]`과 `text-[12px]`이 충돌해요. `href=""`도 페이지 새로고침을 유발해요. (`src/pages/product/list/index.html` 37~79번째 줄)

### 해결 방법

**Step 1.** 충돌하는 `text-[18px]`을 제거하고, Tailwind 유틸리티로 통일하세요.

```html
<!-- 수정 전 (6개 모두 동일) -->
<a href="" class="bg-spring-wood text-[18px] font-normal leading-5 border-[1px] border-empress rounded-[4px] text-woody-brown text-[12px] whitespace-nowrap py-[6px] px-[10px]">

<!-- 수정 후 — 충돌 제거, Tailwind 유틸리티 사용 -->
<a href="#" class="bg-spring-wood text-xs font-normal leading-5 border border-empress rounded text-woody-brown whitespace-nowrap py-1.5 px-2.5">
```

**변경된 부분:**
| 수정 전 | 수정 후 | 이유 |
|---------|---------|------|
| `text-[18px] text-[12px]` | `text-xs` | `text-[18px]` 삭제 (충돌), `text-[12px]` → `text-xs` (Tailwind 기본 12px) |
| `border-[1px]` | `border` | Tailwind `border` 기본이 1px |
| `rounded-[4px]` | `rounded` | Tailwind `rounded` 기본이 4px |
| `py-[6px] px-[10px]` | `py-1.5 px-2.5` | Tailwind 유틸리티 (6px = 1.5 × 4px, 10px = 2.5 × 4px) |
| `href=""` | `href="#"` | 빈 링크 → 새로고침 방지 |

**Step 2.** 6개 `<a>` 태그 모두에 동일하게 적용하세요.

> **쉽게 말하면:** Tailwind에는 이미 `border`, `rounded` 같은 편리한 도구가 있는데, `border-[1px]`, `rounded-[4px]`처럼 직접 수치를 적는 건 "이미 있는 망치 옆에서 손으로 못을 박는 것"과 같아요.

### 해결 방법

**Step 1.** `src/styles/style.css`의 16번째 줄을 수정하세요.

```css
/* 수정 전 (style.css 16번째 줄) */
--color-empress: #7E717;

/* 수정 후 — 6자리로 수정 */
--color-empress: #7E7171;
```

> **중요:** `#7E7171`은 추정값이에요. 디자인 시안(Figma, XD 등)에서 empress 색상의 정확한 코드를 확인하고 넣어주세요. 디자이너에게 물어보는 것이 가장 확실해요!

**Step 2.** 수정 후 개발 서버에서 `border-empress` 클래스가 적용된 요소(상품 목록 카테고리 버튼, 상품 상세 용량 버튼 등)의 테두리가 제대로 보이는지 확인하세요.

> **쉽게 말하면:** 전화번호가 한 자리 빠진 것과 같아요. `010-1234-567`은 전화가 안 걸리는 것처럼, `#7E717`은 색상이 안 나와요.

## 3-10. `--color--grey-99`와 `--color-grey-99`가 중복 정의돼 있어요 (중요도: 중간)

### 문제

하이픈이 2개인 `--color--grey-99`와 1개인 `--color-grey-99`가 동일한 값으로 중복 정의되어 있어요. (`src/styles/style.css` 27, 36번째 줄)

### 해결 방법

**Step 1.** `src/styles/style.css`의 26~27번째 줄을 삭제하세요.

```css
/* 수정 전 (style.css 26~27번째 줄) — 이 두 줄을 삭제 */
/* 로그인,회원가입 배경1(전체배경) */
--color--grey-99: #fffefa;

/* 아래는 유지 (style.css 35~36번째 줄) */
/* 로그인 input */
--color-grey-99: #fffefa; /* 배경색 */
```

**Step 2.** 주석을 업데이트해서 용도를 명확히 하세요.

```css
/* 수정 후 — 주석을 합쳐서 용도를 명확히 */
/* 로그인/회원가입 전체 배경, input 배경색 */
--color-grey-99: #fffefa;
```

> **쉽게 말하면:** 같은 물건에 라벨을 두 개 붙인 것과 같아요. 하나가 오타(`--color--grey-99`)니까 떼어버리고, 올바른 라벨(`--color-grey-99`)만 남기면 돼요.

## 3-11. `style.css`에 같은 색상이 다른 이름으로 중복돼요 (중요도: 낮음)

### 문제

`--color-abbey`와 `--color-grey-abbey`가 같은 값(`#484e51`)이에요. (`src/styles/style.css` 12, 33번째 줄)

### 해결 방법

**Step 1.** `--color-grey-abbey`를 삭제하세요.

```css
/* 수정 전 (style.css 32~33번째 줄) — 이 두 줄을 삭제 */
/* 필수항목,이메일 주소, 비밀번호 글자 색상 */
--color-grey-abbey: #484e51;
```

**Step 2.** 코드에서 `grey-abbey`를 `abbey`로 변경하세요.

`grey-abbey`를 사용하는 곳을 찾아서 `abbey`로 바꿔주세요:

```html
<!-- 수정 전 (login/index.html 34번째 줄) -->
<p class="text-grey-abbey text-[14px] font-normal leading-5">

<!-- 수정 후 -->
<p class="text-abbey text-[14px] font-normal leading-5">
```

```html
<!-- 수정 전 (login/index.html 49번째 줄) -->
<div class="flex flex-col text-grey-abbey text-[14px] ...">

<!-- 수정 후 -->
<div class="flex flex-col text-abbey text-[14px] ...">
```

```html
<!-- 수정 전 (signup/index.html 16번째 줄) -->
<p class="text-grey-abbey text-[12px] ...">

<!-- 수정 후 -->
<p class="text-abbey text-[12px] ...">
```

```html
<!-- 수정 전 (signup/index.html 91번째 줄) -->
<p id="pwGuideText" class="text-grey-abbey text-[12px] ...">

<!-- 수정 후 -->
<p id="pwGuideText" class="text-abbey text-[12px] ...">
```

> **쉽게 말하면:** "철수"라는 이름과 "김철수"라는 이름이 같은 사람을 가리키면, 하나만 쓰기로 정하는 거예요. 둘 다 써도 되지만, 헷갈리니까 하나로 통일하는 게 좋아요.
